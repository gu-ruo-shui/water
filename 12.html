并发编程要处理的问题

  • 通信: 基于共享内存 或 基于消息传递
  • 同步: 各个逻辑之间的前后依赖关系
    • 自旋(忙等待)
    • 阻塞(sleep)

基于共享内存的同步问题：对资源的操作不是原子性

竞态条件成立的三个条件

  • 两个线程共享数据
  • 至少一个处理会对数据进行修改
  • 一个处理未完成之前，另一个介入进来(中间态被打断)

从时间维度上来看, 对资源的操作要保证互斥: 数据的修改操作有且仅有一个程序在活动

共享内存同步

通过 flag 来判断是否有线程对资源的操作，而对 flag 的操作也必须满足原子性，硬件指令来保障 ---> pthread_mutex (futex)

往下一步: 编程需要更方便的指令
condition variable, semaphore(信号量)

void producer() {
    P(&empty);   
    // do sth 假设线程安全
    V(&fill);
}

void consumer() {
    P(&fill);
    // do sth 假设线程安全
    V(&empty);
}

针对业务场景特定性的优化

读读

  • 不需要上锁

写写

  • 必须保证写的顺序性
  • Solution:拆分数据, 线程各自计算(私有数据不需要上锁)，最后再合并(上锁同步)

读写

  • 要么只能看到修改前的数据
  • 要么只能看到全部修改后的数据
  • 不能看到修改的中间态

事务内存

atom {
    // operate(read or write) data code
}

loop:
    data_before := data
    try:
        // operate data code

        if not equal(data_before, data_after):
            raise error
        else:
            commit
    except:
        discard data_after
        goto loop
